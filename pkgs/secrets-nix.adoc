== NAME

secrets.nix - agenix secrets configuration file

== SYNOPSIS

*secrets.nix*

== DESCRIPTION

The *secrets.nix* file defines the secrets managed by *agenix*(1): which public keys can decrypt each secret, whether it has a public counterpart, and how it can be generated. It is a Nix expression that evaluates to an attribute set where each key is a secret name and each value is a secret configuration.

The file is typically located in the same directory as the encrypted secret files, and is referenced by the *--secrets-nix* option or the *SECRETS_NIX* environment variable.

== FILE FORMAT

The file must be a valid Nix expression that evaluates to an attribute set. Each attribute key is the name of a secret, and each value is a secret configuration attribute set.

Secret names are strict:

* No `.age` suffix — the suffix belongs to the file, not the name.
* No path separators — all secret files live in the same directory as `secrets.nix`.
* No leading dot.

Each entry can have up to two files next to `secrets.nix`: the encrypted secret `<name>.age` and the plaintext public counterpart `<name>.pub`.

=== Basic structure

[source,nix]
----
let
  user1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
  system1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
in {
  "secret".publicKeys = [ user1 system1 ];
  "other".publicKeys = [ user1 ];
}
----

This configuration defines two secrets:

* `secret` → stored as `secret.age`
* `other` → stored as `other.age`

== SECRET ATTRIBUTES

Each secret in *secrets.nix* can have the following attributes:

=== publicKeys

List of public keys that can decrypt this secret. Required for every secret that has a secret part. Supports:

* SSH keys: `ssh-ed25519 AAAA...` or `ssh-rsa AAAA...`
* age keys: `age1...`
* References to other secrets' public parts: `"secret-name"`

When a secret name is used as a public key, agenix uses that secret's public part as a recipient, generating it first if necessary.

[source,nix]
----
{
  "secret".publicKeys = [
    "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5..."
    "age1ql3z7hjy54pw3hyww5ayyfg7zqgvc7w3j2elw8zmrj2kg5sfn9aqmcac8p"
    "other-secret"  # Uses other-secret's public part
  ];
}
----

=== hasSecret, hasPublic (optional)

Declare which parts the entry has. The defaults are:

* A name-implied generator (see AUTOMATIC GENERATOR SELECTION) carries its known shape — keypair generators imply `hasPublic = true`.
* Otherwise: `hasSecret = true`, `hasPublic = false`.
* Declaring `hasSecret = false` without declaring `hasPublic` implies a public-only entry (`hasPublic = true`).

Explicit declarations always win. A generator's output must match the effective declaration: a generator that returns a `public` value on an entry whose shape says `hasPublic = false` is an error, as is a missing `secret` on `hasSecret = true`. agenix does not infer the shape from the generator — declare it.

[source,nix]
----
{
  # Public-only entry: no .age file, only metadata in meta.pub
  "meta" = {
    hasSecret = false;
    generator = { }: { public = "metadata-v1"; };
  };

  # Explicit keypair generator: hasPublic must be declared
  "deploy-key" = {
    publicKeys = [ "..." ];
    hasPublic = true;
    generator = { }: builtins.sshKey { };
  };
}
----

=== armor (optional)

When `true`, the encrypted file uses ASCII-armored format instead of binary. Default: `false`.

[source,nix]
----
{
  "secret" = {
    publicKeys = [ "ssh-ed25519 AAAA..." ];
    armor = true;
  };
}
----

=== generator (optional)

A function (or constant value) that produces the secret. *agenix generate* runs it when the entry's files are missing (or with *--force*). Setting `generator = null` explicitly disables generation, including a name-implied generator.

Generators return:

* A string – the secret value
* An attrset with `secret` and/or `public` string values

[source,nix]
----
{
  # Simple generator returning a string
  "api-token" = {
    publicKeys = [ "..." ];
    generator = { }: builtins.randomString 32;
  };

  # Constant value (evaluated once per generation)
  "ssh-key" = {
    publicKeys = [ "..." ];
    hasPublic = true;
    generator = builtins.sshKey { };  # Returns { secret, public }
  };

  # Custom generator with public output
  "my-secret" = {
    publicKeys = [ "..." ];
    hasPublic = true;
    generator = { }: {
      secret = "the-secret-value";
      public = "public-metadata";
    };
  };
}
----

=== dependencies (optional)

List of other secrets this entry is derived from. Dependencies are *regeneration metadata only*: when a listed secret is regenerated, this entry is regenerated as well (unless *--no-dependencies* is given). They are not needed to *read* other secrets in a generator — see GENERATOR FUNCTIONS.

[source,nix]
----
{
  "base-secret" = {
    publicKeys = [ "..." ];
    hasPublic = true;
    generator = { }: builtins.sshKey { };
  };

  # Regenerated whenever base-secret is regenerated
  "derived-secret" = {
    publicKeys = [ "..." ];
    dependencies = [ "base-secret" ];
    generator = { publics }: "Key: ${publics."base-secret"}";
  };
}
----

== GENERATOR FUNCTIONS

A generator function receives the values of *all* entries in `secrets.nix` as lazy attribute sets, and is called with exactly the arguments its pattern names (like `callPackage` in nixpkgs):

*{ }*::
  No arguments needed.

*{ secrets }*::
  Attribute set mapping every entry name to its decrypted secret value.

*{ publics }*::
  Attribute set mapping every entry name to its public part.

*{ secrets, publics }*::
  Both of the above.

All values are lazy: touching `secrets."name"` or `publics."name"` loads, decrypts, or generates that entry on demand. Nothing is resolved for values the generator does not use, resolution order follows the data flow automatically, and circular references are detected and reported.

A generator must be a lambda or a constant value. Bare builtins cannot be called as generators — wrap them:

[source,nix]
----
generator = builtins.sshKey;        # error
generator = { }: builtins.sshKey { };  # correct
generator = builtins.sshKey { };    # also correct: a constant value
----

Generators are evaluated by the *agenix* CLI at generation time, not by Nix itself. The CLI provides additional builtins that are not available in standard Nix.

== BUILT-IN GENERATORS

The following functions are provided by agenix and available as `builtins.*`:

*builtins.randomString* _LENGTH_::
  Random alphanumeric string of the specified length.

*builtins.randomHex* _LENGTH_::
  Random hexadecimal string (lowercase) of the specified length.

*builtins.randomBase64* _BYTES_::
  Random base64-encoded string from the specified number of random bytes.

*builtins.passwordSafe* _LENGTH_::
  Random password using alphanumeric characters plus `-_+=.`.

*builtins.uuid {}*::
  Random UUIDv4 string.

*builtins.sshKey {}*::
  SSH Ed25519 keypair. Returns `{ secret, public }`.

*builtins.rsaKey {}*, *builtins.rsaKey { keySize = N; }*::
  SSH RSA keypair. Optional `keySize`: 2048, 3072, or 4096 (default: 4096). Returns `{ secret, public }`.

*builtins.ageKey {}*::
  age x25519 keypair. Returns `{ secret, public }`.

*builtins.wireguardKey {}*::
  WireGuard keypair (base64). Returns `{ secret, public }`.

*builtins.getSecret* _NAME_, *builtins.getPublic* _NAME_::
  The decrypted secret or public part of another entry. These back the `secrets`/`publics` generator arguments; they are rarely needed directly.

*builtins.blake2b* _STRING_::
  BLAKE2b-512 hash of the string (128 hex characters).

*builtins.blake2s* _STRING_::
  BLAKE2s-256 hash of the string (64 hex characters).

*builtins.keccak* _STRING_::
  SHA3-256 (Keccak) hash of the string (64 hex characters).

== AUTOMATIC GENERATOR SELECTION

When no explicit `generator` is specified, agenix selects one based on the (lowercased) name. The implied generator also implies the entry's shape:

[cols="1,2,1"]
|===
|Name suffix |Generator |Shape

|`ed25519`, `ssh`, `ssh_key`
|SSH Ed25519 keypair
|secret + public

|`x25519`
|age keypair
|secret + public

|`_wg`, `_wireguard`
|WireGuard keypair
|secret + public

|`password`, `passphrase`
|`builtins.randomString 32`
|secret only
|===

== FILES

*secrets.nix*::
  Default configuration file in the current directory. Can be overridden with the *--secrets-nix* option or *SECRETS_NIX* environment variable.

*<secret>.age*::
  Encrypted secret files. The secret name in `secrets.nix` corresponds to these files.

*<secret>.pub*::
  Public counterpart files for entries with `hasPublic = true`.

== EXAMPLES

=== Simple secrets with users and systems

[source,nix]
----
let
  user1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIL0idNvgGiucWgup/mP78zyC23uFjYq0evcWdjGQUaBH";
  system1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPJDyIr/FSz1cJdcoW69R+NrWzwGK/+3gJpqD1t8L2zE";
in {
  "database-password".publicKeys = [ user1 system1 ];
  "api-key".publicKeys = [ user1 ];
}
----

Note that `database-password` automatically gets a `randomString` generator from its name.

=== Generated SSH key with automatic public key file

[source,nix]
----
let
  user1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
  system1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
in {
  # The name suffix implies the SSH keypair generator and hasPublic = true
  "deploy_ed25519".publicKeys = [ user1 system1 ];
}
----

=== Secret referencing another secret's public key

[source,nix]
----
let
  user1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
  system1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
in {
  "deploy_ed25519".publicKeys = [ user1 system1 ];

  # Referencing "deploy_ed25519" in publicKeys encrypts for its public part
  "authorized-keys" = {
    publicKeys = [ user1 system1 "deploy_ed25519" ];
  };
}
----

=== Secret derived from another secret

[source,nix]
----
let
  user1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
  system1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5...";
in {
  "deploy_ed25519".publicKeys = [ user1 system1 ];

  "ssh-config" = {
    publicKeys = [ user1 system1 ];
    # Regenerate ssh-config whenever deploy_ed25519 is regenerated
    dependencies = [ "deploy_ed25519" ];
    generator = { publics }:
      ''
        Host myserver
          IdentityFile /etc/ssh/deploy_key
          # Public: ${publics."deploy_ed25519"}
      '';
  };
}
----

=== Complete configuration example

[source,nix]
----
let
  user1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIL0idNvgGiucWgup/mP78zyC23uFjYq0evcWdjGQUaBH";
  system1 = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPJDyIr/FSz1cJdcoW69R+NrWzwGK/+3gJpqD1t8L2zE";
in {
  # Simple secret, created with `agenix edit` or `agenix encrypt`
  "api-key" = {
    publicKeys = [ user1 ];
    armor = true;
  };

  # Generated from the name: SSH keypair with .pub file
  "host_ed25519".publicKeys = [ user1 system1 ];

  # Generated from the name: random 32-character string
  "service-password".publicKeys = [ user1 system1 ];

  # Explicit keypair generator: declare the shape
  "backup-key" = {
    publicKeys = [ user1 system1 ];
    hasPublic = true;
    generator = { }: builtins.rsaKey { keySize = 4096; };
  };

  # Secret that encrypts for the host key's public part
  "host-token" = {
    publicKeys = [ user1 "host_ed25519" ];
    generator = { }: builtins.randomString 64;
  };

  # Derived secret, regenerated when host_ed25519 rotates
  "known-hosts" = {
    publicKeys = [ user1 system1 ];
    dependencies = [ "host_ed25519" ];
    generator = { publics }: "myserver ${publics."host_ed25519"}";
  };
}
----
